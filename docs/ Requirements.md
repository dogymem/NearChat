**Требования к проекту "NearChat"**

**1 ВВЕДЕНИЕ**

**1.1 Контекст проекта**

Разработка кроссплатформенного мобильного приложения для обнаружения устройств и обмена текстовыми сообщениями в пределах одной локальной сети. Приложение будет использует технологию peer-to-peer (P2P) для установления соединений без необходимости в выделенном сервере. Название продукта: NearChat.

**1.2 Функциональность продукта**

Разрабатываемый продукт должен:

-­ предоставлять возможность общения в реальном времени пользователями, находящимися поблизости в той же сети Wi-Fi;

- ­автоматически обнаруживать других пользователей NearChat в локальной сети;

- ­запоминать никнейм пользователя для идентификации в сети;

- поддерживать базовый обмен текстовыми сообщениями.

Границы разрабатываемого продукта:

- ­работать через интернет;

- ­иметь централизованную систему регистрации и авторизации через внешние сервисы;

- поддерживать передачу файлов, изображений или голосовых сообщений;

- шифровать сообщения сквозным шифрованием.

**2 ТРЕБОВАНИЯ ПОЛЬЗОВАТЕЛЯ**

**2.1 Программные интерфейсы**

Продукт будет разработан с использованием фреймворка SwiftUI, что обеспечит его работу на мобильной операционной системе iOS. Для организации сетевого взаимодействия будет использоваться плагин Network, который абстрагирует работу с Wi-Fi, Bluetooth и геолокацией для обнаружения и подключения к устройствам поблизости.

**2.2 Интерфейс пользователя**

Взаимодействие с пользователем будет осуществлятся через интуитивно понятный мобильный интерфейс.

Основной экран должен состоять из:

- окно сообщений, основную часть экрана должна занимать область переписки;

- поле ввода, в нижней части экрана должно быть расположено текстовое поле для набора нового сообщения и кнопка для его отправки;

- панель навигации, в самом низу экрана должно находится две крупные кнопки, которые будут служить для навигации в приложении.

Также в приложении должен быть экран настроек, содержащий:

- настройки пользователя, с возможностью задать свое имя пользователя;

- режим работы, с возможностью выбрать один из двух доступных режимов работы;

- панель навигации, аналогично основному экрану.

**2.3 Характеристики пользователей**

Продукт ориентирован на пользователей мобильных устройств, которым необходимо быстро организовать общение в локальной сети без доступа к интернету.

Базовый уровень технической грамотности: пользователь, который имеет опыт использования стандартных мобильных мессенджеров, специальные знания не требуются, от пользователя не ожидается понимания сетевых терминов. Процесс обнаружения и подключения должен быть максимально автоматизирован.

**2.4 Предположения и зависимости**

Предположение: все устройства находятся в одной и той же локальной сети Wi-Fi.

Предположение: на устройствах включены Wi-Fi, Bluetooth и службы геолокации для корректной работы механизма обнаружения.

Предположение: сетевые настройки и брандмауэры не блокируют P2P-соединения.

Зависимость: для сборки и запуска проекта требуется установленный Xcode и SwiftUI.

**3 СИСТЕМНЫЕ ТРЕБОВАНИЯ**

**3.1 Функциональные требования**

ФТ-1: система должна позволять пользователю вводить и сохранять свой никнейм.

ФТ-2: система должна загружать сохраненный никнейм при запуске приложения.

ФТ-3: система должна отображать список доступных для чата пользователей.

ФТ-4: после установления соединения пользователи должны иметь возможность отправлять и получать текстовые сообщения в реальном времени.

ФТ-5: все отправленные сообщения должны отображаться у обоих участников чата.

ФТ-6: система должна обрабатывать отключение пользователя, и закрытие сервера.

**3.2 Нефункциональные требования**

НФТ-1 (Производительность): задержка доставки сообщения в локальной сети не должна превышать 1000 миллисекунд.

НФТ-2 (Портируемость): приложение должно работать на ios устройствах.

НФТ-3 (Удобство использования): интерфейс должен быть интуитивно понятным для пользователя мобильных приложений без необходимости в дополнительной документации. Процесс поиска и начала общения должен требовать минимального количества действий.
