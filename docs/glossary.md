# NearChat - Глоссарий ключевых терминов

### AppState (Global State)
Центральное хранилище состояния приложения. Содержит пользовательские настройки (никнейм, IP-адреса, порт) и текущий режим работы, доступные всем экранам через иерархию SwiftUI.

### Base64 Encoding
Метод кодирования данных, используемый в проекте для безопасной передачи никнейма и текста сообщения внутри сетевого пакета. Позволяет избежать ошибок парсинга при использовании специальных символов в тексте чата.

### Broadcast 
Механизм работы сервера, при котором сообщение, полученное от одного клиента, автоматически пересылается всем остальным подключенным участникам чата, чтобы синхронизировать историю переписки у всех пользователей.

### Client (Client Mode)
Режим работы приложения, в котором устройство инициирует исходящее TCP-соединение к указанному IP-адресу и порту сервера для обмена сообщениями.

### ConnectionStatus
Перечисление (`enum`), описывающее текущий этап жизненного цикла сетевого соединения. Включает состояния: ожидание (`idle`), подключение (`connecting`), активное соединение (`connected`/`listening`), ошибка (`failed`) и отключение (`disconnected`).

### NWConnection
Класс фреймворка `Network`, представляющий двунаправленное соединение для передачи данных. Используется как на стороне клиента (для связи с сервером), так и на стороне сервера (для связи с каждым конкретным клиентом).

### NWListener
Класс фреймворка `Network`, отвечающий за прослушивание входящих соединений на указанном порту. Является ядром режима "Сервер".

### ObservableObject / @Published
Механизмы реактивного программирования в Swift. Позволяют `TCPManager` и `AppState` автоматически уведомлять интерфейс (`View`) об изменениях данных, вызывая перерисовку экрана.

### Server (Server Mode / Host)
Режим работы приложения, при котором устройство открывает порт и ожидает входящих подключений от других устройств. В этом режиме приложение выступает в роли координатора беседы.

### TCP (Transmission Control Protocol)
Протокол транспортного уровня, обеспечивающий надежную доставку сообщений в правильном порядке. В проекте используется для стабильного обмена текстовыми данными между устройствами в локальной сети.

### TCPManager
Основной сервис-класс (ViewModel), инкапсулирующий всю сетевую логику. Отвечает за запуск сервера/клиента, отправку сообщений, обработку входящих данных и управление списком активных соединений.

### Message Protocol 
Специфический формат строки данных, используемый для обмена информацией: `timestamp:nickname(Base64):message(Base64)`. Позволяет получателю однозначно разделить время отправки, имя автора и текст сообщения.
